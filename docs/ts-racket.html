How to use ts-racket to automate flyer generation from a course ID.

procedure

(course->flyer course    
  [selling-points    
  registration-link]) → image?
  course : flyer-ready-course?
  selling-points : (listof string?) = (selling-points course)
  registration-link : string? = "https://secure.thoughtstem.com"

Builds a flyer out of one course.

NOTE: alternative spelling of flyer (flier) in this function also works.

Example:
> (scale 0.3 (course->flyer (course 1137)))

procedure

  course2    
  [selling-points    
  selling-points2    
  registration-link]) → image?
  course : flyer-ready-course?
  course2 : flyer-ready-course?

  selling-points2 : (listof string?) = (selling-points course2)
  registration-link : string? = "https://secure.thoughtstem.com"

Builds a flyer out of 2 courses (to be used for two courses at one location).

NOTE: alternative spelling of flyer (flier) in this function also works.

Example:
> (scale 0.3 (courses->flyer (course 1137) (course 1128)))

procedure

(set-room-id resource room-id) → (or/c meeting? course?)

  resource : (or/c meeting? course?)
  room-id : number?

Using this function, you can set the room id for either a singular meeting or all the meetings in a course.

Reminder!! This does not update information in the environment. You must call save or save-meetings! to make changes permanently to the environment.

save-meetings! call (link save) on all the courses’ meetings

save takes a hash keys that are simple values pushed to the set environment

set-env! PROD default is dev
