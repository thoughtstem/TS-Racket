1 General

procedure

(save resource) → resource?

  resource : resource?

Stores the resource on the database (either dev or production, depending on the set-env!

Example:

(save (new-meeting #:course 1000
                   #:start (string->time "2018-09-10 3:30pm")
                   #:duration 60))

procedure

(set-env! k) → void?

  k : dev-or-prod?

Takes either DEV or PROD. Default is DEV.

REMINDER!! Only set to PROD after sufficient testing in DEV.

6 Alerts

procedure

(send-alerts alerts) → void?

  alerts : alerts?

Sends alerts. Alerts should be created with create-alerts. For example, this will send an alert to all computers at meeting 6686.

(send-alerts
 (create-alerts
  (computer-ids (meeting 6686))
  voice-alert
  "Test!"))

procedure

(create-alerts computer-ids    
  alert-type    
  message) → alerts?
  computer-ids : 
(or/c (listof number?)
      string?)
  alert-type : alert-type?
  message : string?

Creates alerts for some list of computer ids. The computer ids can either be a comma separated string (for easy pasting):

"1,2,3"

Or it can be a list of numbers:

'(1 2 3)

procedure

  x : any/c

Can be any member of the following constants.

(define phone-alert   "phone")
(define help-alert    "help")
(define full-alert    "full")
(define block-alert   "block")
(define voice-alert   "voice")
(define message-alert "message")
(define link-alert    "link")
(define command-alert "common")

7 image-util

procedure

(buffer n i) → image?

  n : number?
  i : image?

Adds a transparent "buffer" around an image to assist with spacing and/or printing. First argument is the size in pixels fo the buffer.

Examples:
> (beside
   (square 30 'solid 'green)
   (square 30 'solid 'blue))

> (beside
   (buffer 5 (square 30 'solid 'green))
   (buffer 5 (square 30 'solid 'blue)))

procedure

(scale-to-fit image width) → image?

  image : image?
  width : number?

Scales an image to fit a specific dimention rather than percentage of size.

Example:
> (scale-to-fit logo 100)

procedure

(safe-above imgs ...) → image?

  imgs : image?

An above function that handles being called with 0 or 1 images. Perfect for use in complex image building functions that will be given a range of images and might run into a list of 0 or 1.

procedure

  imgs : image?

A beside function that handles being called with 0 or 1 images. Perfect for use in complex image building functions that will be given a range of images and might run into a list of 0 or 1.

procedure

(safe-overlay imgs) → image?

  imgs : image?

An overlay function that handles being called with 0 or 1 images. Perfect for use in complex image building functions that will be given a range of images and might run into a list of 0 or 1.

procedure

(pad-list imgs) → (listof image?)

  imgs : (listof image?)

Adds transparent images to bring the length of a list up to a number divisible by 3. Build with the purpose of buffering lists of cards being sent to print so they will all scale to the same size no matter how many cards are on the final page.

procedure

(cards->pages imgs) → (listof image?)

  imgs : (listof (or/c image? pict?))

Creates a 3 by 3 page of images (usually used for cards or badges). Ideal for printing!

(cards->pages
 (append
  (badges (course 1208))
  (list
   (build-badge (student 1753) 1208 "Triceratops"))))

procedure

(quest-cards->pages imgs) → (listof image?)

  imgs : (listof pict?)

Like cards->pages, this creates a 3 by 3 page of images. Handles the fact that our quest cards are usually fairly large. Scales them down.

(quest-cards->pages
 quest)

procedure

(print-image! image) → boolean?

Saves an image to a temporary file and uses system commands to print said file.

NOTE: Only works on macs for now.
